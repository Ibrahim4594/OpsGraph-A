# Runbooks

Operational runbooks land here from Milestone 2 onward (telemetry validation, alert triage, rollback procedures). Each runbook is a single markdown file named `<scenario>.md` and is linked from the eventual operator dashboard alongside the recommendation type that triggers it.

Empty in M1 by design — the AIOps core (M2/M3) emits the events that the runbooks describe how to handle.
